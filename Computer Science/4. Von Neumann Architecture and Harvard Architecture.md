# 4. 폰 노이만 구조와 하버드 구조 Von Neumann Architecture and Harvard Architecture

### 4-1. 폰 노이만 구조 (Von Neumann Architecture)

#### 폰 노이만 구조의 개념

폰 노이만 구조(Von Neumann Architecture)는 **프로그램과 데이터를 동일한 메모리에 저장**하고, CPU가 메모리에서 명령어를 순차적으로 가져와 실행하는 컴퓨터 구조이다.

현재 대부분의 범용 컴퓨터가 기본적으로 사용하는 구조이며, **프로그램 내장 방식(Stored Program Concept)** 을 기반으로 한다.

### 4-1. Von Neumann Architecture

#### Concept of Von Neumann Architecture

The Von Neumann Architecture is **a computer architecture that stores programs and data in the same memory and executes instructions by sequentially fetching them from memory through the CPU.**

It is the basic architecture used by most general-purpose computers today and is based on the **Stored Program Concept**.

---

### 프로그램 내장 방식

프로그램 내장 방식은 **프로그램과 데이터를 주기억장치(RAM)에 저장한 후 CPU가 명령어를 순서대로 읽고 실행하는 방식**이다.

이를 통해 하나의 컴퓨터에서 다양한 프로그램을 실행할 수 있으며, 프로그램 수정과 재사용이 가능하다.

### Stored Program Concept

The Stored Program Concept is **a method where programs and data are stored in main memory (RAM), and the CPU reads and executes instructions in sequence.**

This allows a single computer to run various programs and makes program modification and reuse possible.

---

### 폰 노이만 구조의 구성 요소

| 구성 요소 | 역할 |
|---|---|
| CPU | 명령어 해석, 연산, 제어 수행 |
| 주기억장치(RAM) | 프로그램과 데이터 저장 |
| 보조기억장치 | 프로그램과 데이터 영구 저장 |
| 입력장치 | 데이터와 명령어 입력 |
| 출력장치 | 처리 결과 출력 |
| 시스템 버스 | CPU와 장치 간 데이터 전달 |

### Components of Von Neumann Architecture

| Component | Role |
|---|---|
| CPU | Interprets instructions, performs calculations, and controls operations |
| Main Memory (RAM) | Stores programs and data |
| Secondary Storage | Permanently stores programs and data |
| Input Device | Inputs data and instructions |
| Output Device | Outputs processed results |
| System Bus | Transfers data between CPU and devices |

---

### 폰 노이만 구조의 특징

- 프로그램과 데이터를 하나의 메모리에 저장
- 하나의 시스템 버스를 사용
- CPU가 명령어를 순차적으로 실행
- 구조가 단순하고 구현이 쉬움
- 범용 컴퓨터에 적합

### Characteristics of Von Neumann Architecture

- Stores programs and data in the same memory
- Uses a single system bus
- The CPU executes instructions sequentially
- Has a simple structure and is easy to implement
- Suitable for general-purpose computers

---

### 폰 노이만 구조의 장점

- 구조가 단순하여 설계가 쉽다.
- 프로그램 수정과 관리가 편리하다.
- 다양한 프로그램 실행 가능
- 범용 시스템 구축에 적합하다.

### Advantages of Von Neumann Architecture

- Simple structure makes system design easier.
- Program modification and management are convenient.
- Various programs can be executed.
- Suitable for building general-purpose systems.

---

### 폰 노이만 구조의 단점

- 명령어와 데이터를 하나의 버스로 전달한다.
- 명령어와 데이터를 동시에 처리하기 어렵다.
- CPU와 메모리 속도 차이로 성능 저하 발생 가능

이러한 현상을 **폰 노이만 병목(Von Neumann Bottleneck)** 이라고 한다.

### Disadvantages of Von Neumann Architecture

- Instructions and data are transferred through the same bus.
- It is difficult to process instructions and data simultaneously.
- Performance degradation may occur due to the speed gap between CPU and memory.

This phenomenon is called the **Von Neumann Bottleneck**.

### 4-2. 하버드 구조 (Harvard Architecture)

#### 하버드 구조의 개념

하버드 구조(Harvard Architecture)는 **프로그램과 데이터를 서로 다른 메모리에 저장하는 컴퓨터 구조**이다.

프로그램 메모리와 데이터 메모리를 분리하여 각각 별도의 버스를 사용하기 때문에 명령어와 데이터를 동시에 처리할 수 있다.

### 4-2. Harvard Architecture

#### Concept of Harvard Architecture

The Harvard Architecture is **a computer architecture that stores programs and data in separate memories.**

Since program memory and data memory are separated and use independent buses, instructions and data can be processed simultaneously.

---

### 하버드 구조의 구성 요소

| 구성 요소 | 역할 |
|---|---|
| CPU | 명령어 실행 및 데이터 처리 |
| 프로그램 메모리 | 명령어 저장 |
| 데이터 메모리 | 데이터 저장 |
| 프로그램 버스 | 명령어 전달 |
| 데이터 버스 | 데이터 전달 |
| 입력장치 | 데이터 입력 |
| 출력장치 | 결과 출력 |

### Components of Harvard Architecture

| Component | Role |
|---|---|
| CPU | Executes instructions and processes data |
| Program Memory | Stores instructions |
| Data Memory | Stores data |
| Program Bus | Transfers instructions |
| Data Bus | Transfers data |
| Input Device | Inputs data |
| Output Device | Outputs results |

---

### 하버드 구조의 특징

- 프로그램과 데이터를 별도의 메모리에 저장
- 명령어 버스와 데이터 버스를 분리
- 명령어와 데이터를 동시에 처리 가능
- 높은 처리 속도 제공

### Characteristics of Harvard Architecture

- Stores programs and data in separate memories
- Separates instruction bus and data bus
- Can process instructions and data simultaneously
- Provides high processing speed

---

### 하버드 구조의 장점

- 명령어와 데이터를 동시에 읽을 수 있다.
- 메모리 접근 병목 현상이 감소한다.
- 실시간 처리가 필요한 시스템에 적합하다.

### Advantages of Harvard Architecture

- Instructions and data can be read at the same time.
- Memory access bottlenecks are reduced.
- Suitable for systems requiring real-time processing.

---

### 하버드 구조의 단점

- 구조가 복잡하다.
- 설계 비용이 증가한다.
- 메모리 관리가 어렵다.

### Disadvantages of Harvard Architecture

- The structure is complex.
- Design costs increase.
- Memory management is more difficult.

---

### 4-3. 폰 노이만 구조와 하버드 구조 비교

| 구분 | 폰 노이만 구조 | 하버드 구조 |
|---|---|---|
| 메모리 구조 | 프로그램과 데이터를 같은 메모리에 저장 | 프로그램과 데이터를 분리 저장 |
| 버스 구조 | 하나의 버스 사용 | 별도의 버스 사용 |
| 처리 방식 | 순차 처리 | 병렬 처리 가능 |
| 속도 | 상대적으로 느림 | 상대적으로 빠름 |
| 구조 | 단순 | 복잡 |
| 비용 | 낮음 | 높음 |
| 활용 분야 | PC, 노트북, 서버 | 임베디드 시스템, MCU |

### 4-3. Comparison of Von Neumann and Harvard Architectures

| Category | Von Neumann Architecture | Harvard Architecture |
|---|---|---|
| Memory Structure | Stores programs and data in the same memory | Stores programs and data separately |
| Bus Structure | Uses a single bus | Uses separate buses |
| Processing Method | Sequential processing | Supports parallel processing |
| Speed | Relatively slower | Relatively faster |
| Structure | Simple | Complex |
| Cost | Lower | Higher |
| Applications | PCs, laptops, servers | Embedded systems, MCUs |

---

### 4-4. 현대 컴퓨터 구조

현대 CPU는 순수한 폰 노이만 구조가 아니라 **수정된 하버드 구조(Modified Harvard Architecture)** 를 사용한다.

수정된 하버드 구조는 폰 노이만 구조의 장점인 범용성을 유지하면서, 하버드 구조의 장점인 빠른 데이터 처리를 적용한 방식이다.

대표적인 방식은 CPU 내부의 캐시를 분리하는 것이다.

- 명령어 캐시 (Instruction Cache)
- 데이터 캐시 (Data Cache)

이를 통해 CPU는 명령어와 데이터를 동시에 처리하여 성능을 향상시킨다.

### 4-4. Modern Computer Architecture

Modern CPUs do not use a pure Von Neumann Architecture. Instead, they use a **Modified Harvard Architecture**.

The Modified Harvard Architecture maintains the general-purpose advantages of the Von Neumann Architecture while applying the fast data processing advantages of the Harvard Architecture.

A common implementation is separating the cache inside the CPU.

- Instruction Cache
- Data Cache

Through this method, the CPU can process instructions and data simultaneously, improving performance.

---

### 핵심 정리

- 폰 노이만 구조는 **프로그램과 데이터를 같은 메모리에 저장하는 구조**
- 하버드 구조는 **프로그램과 데이터를 분리하여 저장하는 구조**
- 폰 노이만 구조는 범용성이 높고, 하버드 구조는 처리 속도가 빠르다.
- 현대 CPU는 두 구조의 장점을 결합한 **수정된 하버드 구조**를 사용한다.

### Key Summary

- The **Von Neumann Architecture** stores programs and data in the same memory.
- The **Harvard Architecture** stores programs and data in separate memories.
- The Von Neumann Architecture provides high versatility, while the Harvard Architecture provides faster processing speed.
- Modern CPUs use a **Modified Harvard Architecture** that combines the advantages of both architectures.
